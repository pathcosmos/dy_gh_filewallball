🎯 API 개요

📝 FileWallBall Simplified API

복잡한 인증과 기능들을 제거하고 핵심 파일 관리 기능만 제공하는 간소화된 API입니다.

🚀 주요 개선사항

  • 단순화: 20+ 엔드포인트 → 7개 핵심 엔드포인트
  • 인증 제거: 프로젝트 키, Bearer 토큰 불필요
  • 직관적 URL: /api/v1/files/ → /files/, /upload
  • 빠른 처리: 복잡한 서비스 레이어 제거
  • 안정성: 기본적인 파일 검증과 해시 계산

🔧 기술 스택

  • Framework: FastAPI + SQLAlchemy
  • Database: MariaDB (또는 SQLite)
  • Storage: 로컬 파일시스템 (/app/uploads)
  • Security: 기본 파일 크기/타입 검증
  • Performance: 스트리밍 업로드/다운로드

📋 지원 기능

  • 파일 업로드 (최대 100MB)
  • 파일 다운로드 (스트리밍)
  • 파일 정보 조회
  • 파일 목록 (페이지네이션)
  • 텍스트 파일 미리보기
  • 파일 삭제
  • SHA-256 해시 기반 무결성 검증

🛠️ API 엔드포인트

🎉 모든 엔드포인트는 인증 없이 사용 가능

간편하고 빠른 파일 관리를 위해 인증 과정을 제거했습니다.

기능 메서드 경로 파라미터 설명
Health Check GET /health - API 상태 확인
파일 업로드 POST /upload file UploadFile 필수
form-data로 전송 		파일 업로드 (최대 100MB)
파일 정보 GET /files/{file_id} file_id string (UUID) 필수 파일 메타데이터 조회
파일 다운로드 GET /download/{file_id} file_id string (UUID) 필수 파일 다운로드 (스트리밍)
파일 미리보기 GET /view/{file_id} file_id string (UUID) 필수 텍스트 파일 내용 미리보기 (1000자)
파일 목록 GET /files limit int = 10 선택
offset int = 0 선택 		파일 목록 조회 (페이지네이션)
파일 삭제 DELETE /files/{file_id} file_id string (UUID) 필수 파일 완전 삭제 (파일시스템 + DB)

💻 사용 예제

1. 파일 업로드

# cURL 예제 curl -X POST http://localhost:8000/upload \ -F "file=@example.txt" # Python requests 예제 import requests with open('example.txt', 'rb') as f: response = requests.post('http://localhost:8000/upload', files={'file': f}) print(response.json())
응답:
{ "file_id": "123e4567-e89b-12d3-a456-426614174000", "filename": "example.txt", "file_size": 1024, "upload_time": "2024-01-15T10:30:00", "download_url": "/download/123e4567-e89b-12d3-a456-426614174000", "view_url": "/view/123e4567-e89b-12d3-a456-426614174000", "message": "File uploaded successfully" }

2. 파일 정보 조회

# cURL 예제 curl http://localhost:8000/files/123e4567-e89b-12d3-a456-426614174000 # Python requests 예제 response = requests.get('http://localhost:8000/files/123e4567-e89b-12d3-a456-426614174000') print(response.json())
응답:
{ "file_id": "123e4567-e89b-12d3-a456-426614174000", "filename": "example.txt", "file_size": 1024, "mime_type": "text/plain", "upload_time": "2024-01-15T10:30:00", "file_hash": "a665a45920422f9d417e4867efdc4fb8a04a1f3fff1fa07e998e86f7f7a27ae3" }

3. 파일 목록 조회

# 기본 목록 (10개) curl http://localhost:8000/files # 페이지네이션 curl "http://localhost:8000/files?limit=20&offset=10"

4. 파일 다운로드

# 브라우저에서 직접 접근 http://localhost:8000/download/123e4567-e89b-12d3-a456-426614174000 # cURL로 다운로드 curl -O -J http://localhost:8000/download/123e4567-e89b-12d3-a456-426614174000

5. 텍스트 파일 미리보기

curl http://localhost:8000/view/123e4567-e89b-12d3-a456-426614174000
응답 (텍스트 파일인 경우):
{ "file_id": "123e4567-e89b-12d3-a456-426614174000", "filename": "example.txt", "content": "This is the content of the file...", "truncated": false }

6. 파일 삭제

curl -X DELETE http://localhost:8000/files/123e4567-e89b-12d3-a456-426614174000

📋 응답 형식 및 상태 코드

HTTP 상태 코드

상태 코드 설명 발생 상황
200 성공 요청이 성공적으로 처리됨
201 생성됨 파일 업로드 성공
400 잘못된 요청 파일 없음, 잘못된 파라미터
404 찾을 수 없음 파일이 존재하지 않음
413 파일 크기 초과 100MB 제한 초과
500 서버 오류 내부 서버 오류

에러 응답 형식

{ "detail": "Error message here" }

⚠️ 파일 크기 제한

업로드 파일 크기는 최대 100MB로 제한됩니다.

🔒 보안 고려사항

  • 파일 확장자 기반 기본 검증
  • SHA-256 해시를 통한 파일 무결성 확인
  • 스트리밍을 통한 메모리 효율적 처리
  • UUID 기반 파일 식별로 예측 불가능한 URL
